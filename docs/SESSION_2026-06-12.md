## Decisões arquiteturais reforçadas

1. **Domain types abstratos** sobre enums externos. Um único módulo
   conhece a forma crua (`biddingLearning.ts:classifyBiddingLearning`).
   Rest of code fala domain. Se Google renomear enum em v21, só
   classifier muda.

2. **Soft caps sempre downgradam para `needs_human_review`**, nunca
   `blocked`. Recuperar de bloqueio rígido erradamente disparado custa
   intervenção out-of-band; `needs_human_review` preserva segurança sem
   rigidez.

3. **Fail-closed em precondições do executor.** Em vez de assumir e
   torcer, retornar 422 com motivo estruturado.

4. **Fake DB extensível.** Quando a query precisa de operador novo,
   estendemos `_fakeDb.ts` em vez de driblar a feature.
   Suporta agora: `=`, `IN (...)`, `IS NULL`, `IS NOT NULL`, `>`, `>=`,
   `<`, `<=`, `ALTER TABLE` (no-op).

5. **Composição via `mergeVerdicts`.** Toda guardrail nova retorna
   `GuardrailVerdict | null` e é mergeada por severidade. Adicionar
   nova guardrail = novo módulo + 1 linha no refine.
